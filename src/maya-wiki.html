What is Maya?

Maya is a standalone package manager written in bash, which follows the same scheme as ports, for example.
It uses pre-built recipes to download, check dependencies, compile and install binaries
And, if removal is necessary, it also removes the binaries and all traces they leave behind.

You can find the source code for maya here.

Installation:

To install from source code, you will need the following dependencies:
    coreutils, gcc, git, wget, tar
First, we will check if the following two directories exist:
    $ ls $HOME/.local/{bin,share}
If you receive an error message indicating that directories do not exist, create them (if they already exist, ignore this step):
    $ mkdir -p $HOME/.local/{share,bin}
Next, we will clone the remote repository to ~/.local/share/maya.
    $ git clone https://github.com/sakaki91/maya $HOME/.local/share/maya
    $ ln -sf $HOME/.local/share/maya/maya $HOME/.local/bin/
If you don't have $HOME/.local/bin added to your $PATH in your .bashrc (or the shell you use), add it with:
    $ echo 'export PATH="$HOME/.local/bin:$PATH"' >> "$HOME"/.bashrc
After that, close the terminal and open it again, or reopen your shell interpreter.

Use:

    $ maya
    :: usage: maya --help

First steps:

To see all the arguments that can be used, run:
    $ maya -h
    operations:
    -h  | --help             :: show this message.
    -v  | --version          :: shows the version.
    -a  | --about            :: view the metadata (version, name, and description) of a package.
    -l  | --list             :: shows all binaries available for installation.
    -li | --list-installed   :: it shows everything that maya has installed.
    -cl | --clean            :: clears the /tmp/maya directory.
    -sy | --sync-maya        :: update maya and recipes by pulling directly from the git.
    -in | --install          :: install a binary.
    -rm | --remove           :: remove a binary.

    usage: maya 'operation' [...]
As you can see, everything is very legible and easy to understand.
The next modules will provide some examples of how to use these modules.

Function '-a | --about'

The '-a --about' function lists the metadata of the packages present in the $HOME/.local/share/maya/pkg/ directory for installation, for example:
    $ maya -a make
    make :: 4.4.1             GNU make source code.
    
    $ maya -a discord
    discord :: 1.0.149        All-in-one voice and text chat for gamers that's free, secure, and works on both your desktop and phone.

Function '-l | --list'

The function '-l | --list' lists all recipes contained in $HOME/.local/share/maya/pkg/ that are available for installation, for example:
    $ maya -l
    cbonsai
    cmake
    discord
    fastfetch
    [...]
There is also a filtering option within the '-l | --list' function:
    $ maya -l fastfetch
    fastfetch
    neofetch
    onefetch
    pfetch

Function '-li | --list-installed'

The function '-li | --list-installed' follows the same scheme as the previous function, but filters what was installed by the maya package manager.

When installing a binary, maya creates a file in $HOME/.local/share/maya/pkg/info/ to serve as a way to "track" what is installed.

After installing the desired binaries, running 'maya -li' will list what was tracked here previously, for example:
    $ maya -li 
    discord
    fastfetch
    make
    onefetch
    vault
And the "tracking" directory contains the following files:
    $ ls $HOME/.local/share/maya/pkg/info/
    discord  fastfetch  make  onefetch  vault

Function '-cl | --clean'

When a binary is installed using Maya, Maya creates a temporary directory (/tmp/maya) precisely to "centralize" the tarballs/source-codes of the binaries.
The function '-cl | --clean' serves precisely to clean this entire folder, for example:
    $ ls /tmp/maya/
    example1  example2  example3

    $ maya --cl
    [sudo] password:

    $ ls /tmp/maya
    ls: could not access '/tmp/maya': File or directory does not exist

Function '-sy | --sync-maya'

The function '-sy | --sync-maya' is used to update maya, pulling its new changes directly from Git, along with ALL the new package recipes added.

Daily use of this function is recommended to always have the largest number of packages available for installation/compilation, example:
    $ maya -sy
    Already up to date.

Function '-in | --install'

The '-in | --install' function is one of the most important functions in Maya, *obviously*.
Maya uses a philosophy and style somewhat reminiscent of FreeBSD Ports, but with an EXTREME and RELENTLESS focus on tracking orphaned packages/files during installations and removals.

And, obviously, following the style of *Ports*, the installation of binaries/programs will depend heavily on your hardware, precisely because ABSOLUTELY EVERYTHING IS COMPILED FROM SCRATCH, there are no ready-made repositories or binaries, only predefined recipes that Maya can read.

Below is an example of an installation:
    $ maya -in pfetch
The standard pfetch recipe below is shown as an example:
    $ cat $HOME/.local/share/maya/pkg/pfetch
    #!/bin/bash

    NAME="pfetch"
    VERSION="0.6.0"
    DESCRIPTION="A pretty system information tool written in POSIX sh."
    SOURCE='https://github.com/dylanaraps/pfetch'

    function install(){
        git clone $SOURCE /tmp/maya/$NAME
        sudo cp /tmp/maya/$NAME/$NAME /usr/bin/$NAME
    }

    function remove(){
       sudo rm -rf /bin/$NAME "$MAYA_TMP/$NAME"
    }
This example was used precisely because of its simplicity; however, more complex projects like make have more complex recipes, for example:
    $ cat $HOME/.local/share/maya/pkg/make
     
    #!/bin/bash

    NAME="make"
    VERSION="4.4.1"
    DESCRIPTION="GNU make source code."
    SOURCE="https://ftp.gnu.org/gnu/$NAME/$NAME-$VERSION.tar.gz"
    DEPENDENCIES=(
        "wget"
        "tar"
        "gcc"
    )

    function install(){
        wget -c $SOURCE --directory-prefix=/tmp/maya/$NAME
        tar -xvf /tmp/maya/$NAME/$NAME-$VERSION.tar.gz --directory=/tmp/maya/$NAME
        (
            cd /tmp/maya/$NAME/$NAME-$VERSION
            sudo ./configure --disable-dependency-tracking
            sudo ./build.sh
            sudo ./make
            sudo ./make install

This means that ANYONE can contribute if they want, without actually needing a ready-made repository, as long as everything is filled in correctly.
If you are interested, there is a package-template.sh file in ~/.local/share/maya/src/package-template.sh

Function '-rm | --remove'

The '-rm | --remove' function is also extremely important, as it performs a RELENTLESS cleanup of ALL POSSIBLE ORPHAN FILES from the desired binary, for example:
    $ maya -rm pfetch
Remember that both '-in' and '-rm' only work for files that were installed/removed by maya, as they have their own internal registry $HOME/.local/maya/pkg/info

Uninstallation:

Removing maya is relatively straightforward because, just as I follow an extremely paranoid philosophy regarding orphaned files scattered throughout the file system, ALL of maya (if installed correctly following the Wiki) has ALL its main files centralized in $HOME/.local/share/maya, with a temporary directory in /tmp/maya, and a symbolic execution link in $HOME/.local/bin/maya'.

In short, to completely remove Maya, simply run:
    $ rm -rf $HOME/.local/share/maya $HOME/.local/bin/maya /tmp/maya
I STRONGLY recommend that in cases of uninstallation, you also remove the packages installed by Maya, as it will not be possible to maintain them (unless it is intentional).
